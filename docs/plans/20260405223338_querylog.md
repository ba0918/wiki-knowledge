## What & Why

wiki-query 実行時にクエリのメタデータを構造化ログとして蓄積する。
これは Phase 2+ の全機能（Trust Score、Gap Detection、Auto Ingest 提案）の基盤となるデータレイヤー。

現在の wiki-query は `log.md` に1行ログと `outputs/queries/` にマークダウン回答を保存しているが、
**機械的に集計可能な構造化データ**がない。QueryLog はこのギャップを埋める。

## Goals

- query 実行のたびに構造化メタデータを `{wiki_root}/outputs/querylog.jsonl` に追記する
- 既存の query フロー（log.md 追記、outputs/queries/ 保存）を壊さない
- 後続機能（Trust Score 集計、Gap Detection 分析）が QueryLog を入力として使える形式にする
- QueryLog の集計・閲覧ユーティリティを提供する

## Design

### QueryLog エントリ形式（JSONL）

```jsonl
{"id":"q_20260405T223000","timestamp":"2026-04-05T22:30:00+09:00","question":"Ingest と Compile の違いは？","sources_consulted":["concepts/llm-wiki-knowledge-base.md","concepts/wiki-knowledge-architecture.md"],"sources_cited":["concepts/llm-wiki-knowledge-base.md"],"gap_noted":false,"gap_topics":[],"promoted":false,"promoted_to":null}
```

### フィールド定義

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `id` | string | `q_{YYYYMMDDTHHMMSS}` 形式（タイムスタンプベース、read-before-write 不要） |
| `timestamp` | string | ISO 8601 タイムスタンプ |
| `question` | string | ユーザの質問文（全文） |
| `sources_consulted` | string[] | index スキャンで読み込んだ記事パス |
| `sources_cited` | string[] | 回答テキストから `[[wikilink]]` を正規表現抽出して収集（`re.findall(r"\[\[([a-z0-9-]+)\]\]", response)` 相当） |
| `gap_noted` | boolean | 回答中に「Wiki にない情報」を指摘したか |
| `gap_topics` | string[] | ギャップとして指摘したトピック（自由テキスト） |
| `promoted` | boolean | 回答を concepts/ に昇格したか |
| `promoted_to` | string\|null | 昇格先のパス（promoted=true の場合） |

### ID 採番ルール

タイムスタンプベースの `q_{YYYYMMDDTHHMMSS}` を使用する。seq 番号方式（`q_20260405_001`）は read-before-write が必要で脆弱なため不採用。同一秒の衝突は実運用上無視できる（人間が手動で query を実行するため）。

### sources_cited の抽出方法

回答テキストから `[[wikilink]]` パターンを正規表現で抽出する（方式 A）。プロンプトで別途リストを出力させる方式 B は、回答フォーマットが複雑になるため不採用。既存の `find_wikilinks()` 関数（lint-wiki.py）と同じロジックを再利用する。

### querylog.jsonl の初回作成

ファイルが存在しない場合、最初の query 実行時に自動作成する（append モード `>>` で書き込めば存在チェック不要）。空ファイルを wiki-init で事前作成はしない（outputs/ 配下は実行時に動的生成するポリシー）。

### git 管理ポリシー

`querylog.jsonl` はユーザの質問文を含むため、**デフォルトでは git 管理対象外**とする。

- wiki-init で生成する `.wiki/.gitignore` に `outputs/querylog.jsonl` を追加する
- ユーザが明示的に git 管理したい場合は `.gitignore` から除外できる
- SKILL.md に「querylog.jsonl には質問文が記録される」旨の注意書きを記載する

### 変更対象ファイル

```
skills/wiki/
├── SKILL.md                          # query セクションに QueryLog 追記手順を追加
├── references/
│   ├── frontmatter-schemas.md        # QueryLog エントリスキーマを追加
│   └── prompts.md                    # query プロンプトに gap_topics 抽出指示を追加
└── scripts/
    └── querylog-stats.py             # 新規: QueryLog 集計スクリプト

.wiki/
└── outputs/
    └── querylog.jsonl                # 新規: QueryLog データファイル
```

### query フローへの統合ポイント

現在のフロー（変更なし）:
```
1. Index スキャン → 関連記事特定
2. 記事読み込み
3. 回答合成（[[wikilink]] 引用付き）
4. 保存提案 → outputs/queries/ or concepts/ に保存
5. log.md 追記
```

追加するステップ:
```
3.5. 回答合成時に gap_topics を抽出（プロンプトに組み込み）
5.5. QueryLog エントリを outputs/querylog.jsonl に追記
```

### querylog-stats.py の機能

```bash
# 基本統計（--wiki-root で concepts/ を参照して未参照記事を検出）
python3 querylog-stats.py --wiki-root .wiki

# 出力例:
# Total queries: 42
# Unique sources consulted: 8/12 (66.7%)
# Sources never consulted: concept-a.md, concept-b.md, ...
# Gap noted: 15/42 (35.7%)
# Top gap topics: "RAG architecture" (5), "embedding models" (3), ...
# Promoted: 3/42 (7.1%)
```

スクリプトは `{wiki_root}/outputs/querylog.jsonl` を自動的に読み込む。`concepts/` と突き合わせて未参照記事を検出するため、`--wiki-root` が必須引数。

### Key Points

- **Append-only**: querylog.jsonl は追記のみ。編集・削除はしない
- **既存フローに影響なし**: log.md と outputs/queries/ は従来通り。QueryLog は追加のデータレイヤー
- **LLM が書く**: querylog-stats.py は人間用の集計ツール。LLM は JSONL を直接読んで分析する
- **gap_topics が Gap Detection の種**: query 時に「Wiki にない」と指摘した内容を構造化して貯める
