## 目的

Wiki 記事ごとに「どの程度信頼できるか」を定量評価するスコアを算出する。
記事の品質を可視化し、lint / query / compile の判断材料として活用する。

## Trust Score 計算式

### 4つの要素（0.0〜1.0 に正規化して加重合計）

| 要素 | 重み | 算出方法 | 根拠 |
|------|------|----------|------|
| **ソース数** | 0.30 | `len(source_refs)` を正規化 | 複数ソースで裏付けされた記事は信頼性が高い |
| **鮮度** | 0.20 | `updated` 日付からの線形減衰（365日で 0.0） | 古い記事は陳腐化リスク |
| **引用頻度** | 0.30 | QueryLog の `sources_cited` での出現回数を正規化 | 実際に引用される記事は有用性が高い |
| **backlink数** | 0.20 | 他記事の `related` + 本文 `[[wikilink]]` での被参照数を正規化 | ハブ記事は構造的に重要 |

### 各要素の raw value 定義

| 要素 | raw value | 説明 |
|------|-----------|------|
| ソース数 | `len(source_refs)` | フロントマターの source_refs 配列の長さ |
| 鮮度 | `max(0.0, 1.0 - elapsed_days / 365)` | 線形減衰。updated から365日で 0.0 に到達。365日超は 0.0 |
| 引用頻度 | QueryLog の `sources_cited` に含まれる回数 | slug 正規化後にカウント |
| backlink数 | 他記事からの被参照数（**deduplicated**） | 同一記事から `related` + `[[wikilink]]` で重複参照しても **1回** とカウント。slug 正規化: `related` の `concepts/foo.md` → `foo`、`[[foo]]` → `foo` で統一後に set で deduplicate |

### 正規化

- 各要素を `min-max` で 0.0〜1.0 に正規化
- **記事数 < 3 の場合**: min-max が不安定なため、全要素の正規化値を **0.5 固定**（中央値）とする
- **最終スコア**: `0.30 * source + 0.20 * freshness + 0.30 * citation + 0.20 * backlink`
- スコア範囲: 0.0〜1.0（小数第2位まで）

### QueryLog が空の場合

QueryLog エントリが0件の場合、引用頻度を除外して残りの3要素で再配分:

| 要素 | 重み（フォールバック） |
|------|----------------------|
| ソース数 | 0.40 |
| 鮮度 | 0.30 |
| backlink数 | 0.30 |

## ステップ

### Step 1: trust_score.py — スコア計算スクリプト

**ファイル**: `skills/wiki/scripts/trust_score.py`

純粋関数ベースの計算エンジン。CLI としても使える。

```
入力:
  - concepts/*.md のフロントマター（source_refs, updated, related）
  - concepts/*.md の本文（wikilink 抽出）
  - outputs/querylog.jsonl（任意、なくても動く）

出力:
  - 各記事の Trust Score（JSON or テーブル）
  - スコア内訳（4要素の個別値）
```

**実装詳細**:
- `compute_trust_scores(articles, querylog_entries) -> list[ArticleScore]`
  - 純粋関数。副作用なし。
- `parse_article_metadata(concept_dir) -> list[ArticleMeta]`
  - フロントマター + wikilink 抽出
- `count_backlinks(articles) -> dict[str, int]`
  - 各記事が他記事から何回参照されているか
- `count_citations(entries, articles) -> dict[str, int]`
  - QueryLog の sources_cited から引用回数を集計
- `normalize_scores(raw_scores) -> list[float]`
  - min-max 正規化
- CLI: `python3 skills/wiki/scripts/trust_score.py --wiki-root .wiki [--format table|json]`

### Step 2: test_trust_score.py — テスト

**ファイル**: `skills/wiki/scripts/test_trust_score.py`

- 純粋関数のユニットテスト
- エッジケース: 記事0件、QueryLog空、全記事同一スコア、source_refs=1 vs 多数
- フォールバック重み配分のテスト
- 正規化の境界値テスト

### Step 3: SKILL.md lint ワークフロー統合

**ファイル**: `skills/wiki/SKILL.md` の lint セクションを更新

- `lint-wiki.py` は変更しない（構造チェックに特化を維持、単一責任原則）
- SKILL.md の lint ワークフローで「lint-wiki.py 実行後に trust_score.py も実行」とオーケストレーション
- スコアが閾値（0.3）未満の記事を lint レポートの 🟡 Warning として記載

### Step 4: レポート出力

**出力先**: `{wiki_root}/outputs/reports/{YYYYMMDD}-trust-score.md`

- 既存の lint レポートと同パターンで Output 層に永続化
- 記事一覧をスコア降順で表示、内訳（4要素）付き
- CLI の `--format` オプション: `table`（デフォルト）/ `json` / `report`（Markdown ファイル出力）

### Step 5: ドキュメント更新

- SKILL.md の query/lint セクションに Trust Score の言及を追加
- `skills/wiki/references/frontmatter-schemas.md` は変更不要（フロントマターに trust_score を追加しないため）
- CLAUDE.md に Trust Score の説明を追加

> **スコープ外（Phase 3b で検討）**: query 回答時の低スコア注釈。現在5記事で QueryLog も初期状態のため、運用データが溜まってから統合する。

## リスク & 対策

| リスク | 対策 |
|--------|------|
| QueryLog が空の初期状態でスコアが無意味 | フォールバック重み配分で 3要素のみで計算 |
| 記事数が少ない（現在5件）で正規化が不安定 | 記事数 < 3 の場合は全要素の正規化値を 0.5 固定 |
| source_refs のパスが壊れている可能性 | lint の既存 missing source チェックと連携 |
| 重み配分の妥当性が不明 | 設定ファイル化は YAGNI。まず固定値で運用し、フィードバックで調整 |
